# CLAUDE.md — Imberion Website Rules

## Always First
- Invoke `frontend-design` skill before writing frontend code.
- Read this file before making changes.

## Core Goal
Build a clear, premium, branded Imberion website.

The site must quickly communicate:
1. What it is
2. What problem it solves
3. Who it is for
4. What to do next

Prioritize clarity and low cognitive load.

## Working Mode
For non-trivial tasks:
- Start with a plan
- Restate objective
- Define problem
- Outline approach
- Ask if unclear

## Verification
Before:
- Explain how success will be verified

After:
- Check against plan
- Check side effects
- Confirm clarity, hierarchy, usability improved

Use screenshots or inspection when needed.

## Brand & Design
- Check `brand_assets/` first
- Follow `BRAND_GUIDELINES.md` if exists
- Use existing logo, colors, typography
- Preserve current design direction unless told otherwise

## References
- If reference image exists → match layout, spacing, hierarchy
- Do not “improve” unless asked

## Dev Rules
- Work on localhost only
- Do not start duplicate servers

## Git / Deploy
- Do not push or deploy unless explicitly instructed
- Always test locally first

## Hard Rules
- No filler sections
- No new brand direction
- No generic SaaS defaults
- Do not use `transition-all`
